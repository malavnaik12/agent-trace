AgentTrace — Technical Spec
Research & Synthesis Agent · LangGraph · Audit Layer · PostgreSQL Schema
01 What we're building

A research & synthesis agent that takes a question, searches the web, fetches and classifies content, and produces a grounded answer. The agent itself is the demo. The audit layer is the differentiator — every decision, tool call, and state transition is recorded, diffed, and inspected for anomalies.

Every node in the graph is wrapped by a tracing decorator that creates a structured span — analogous to an OpenTelemetry span but persisted to Postgres and queryable. A lightweight anomaly detector flags deviations from expected behavior. A FastAPI layer exposes the trace store. A dashboard visualises the audit trail per run.

The core thesis: production agentic AI needs the same assurance primitives as safety-critical embedded systems — observable state, traceable decisions, anomaly detection. This project builds a minimal but complete version of that stack.
02 Agent state definition

The state is the single source of truth for the entire run. Every node reads from it and writes to it. The audit layer diffs the state before and after each node execution — the diff is what gets stored as evidence of what the node actually did.

pythonagent/state.py
from typing import TypedDict, List, Optional, Annotated
from dataclasses import dataclass, field
import operator

# Immutable result types — passed through state as lists
@dataclass
class SearchResult:
    query: str
    url: str
    title: str
    snippet: str

@dataclass
class FetchedPage:
    url: str
    content: str          # raw extracted text, max 8000 chars
    fetch_success: bool

@dataclass
class RelevantChunk:
    url: str
    content: str
    relevance_score: float  # 0.0 – 1.0, assigned by classify node
    kept: bool              # True if score >= threshold


# The full agent state — persisted as JSONB snapshots at each node
class AgentState(TypedDict):
    # Identity
    run_id:         str                    # UUID, set at graph entry
    query:          str                    # original user question

    # Planning output
    sub_queries:    List[str]              # decomposed search queries

    # Search + fetch output
    search_results: List[SearchResult]    # raw results from web search
    fetched_pages:  List[FetchedPage]     # full page content

    # Classification output
    relevant_chunks:List[RelevantChunk]   # scored + filtered content

    # Synthesis output
    draft_answer:   str                    # pre-verification answer
    final_answer:   str                    # verified, returned to user

    # Control flow
    iteration:      int                    # how many search loops so far
    verified:       bool                   # did verify node accept the answer?
    verification_feedback: Optional[str]  # why verify rejected (if it did)
03 LangGraph architecture

Six nodes. One conditional edge (verify → search or END). Max two iterations enforced in the routing function to prevent unbounded loops — which is itself one of the anomalies we detect.

plan
LLM call. Decomposes the user query into 2–3 specific search sub-queries. Writes to sub_queries.
search
Tool call (Tavily or SerpAPI). Runs each sub-query, collects top-3 results. Writes to search_results.
fetch
HTTP tool call. Fetches and strips HTML for top URLs. Caps at 8000 chars/page. Writes to fetched_pages.
classify
LLM call. Scores each page chunk 0–1 against the original query. Filters below 0.5. Writes to relevant_chunks.
synthesize
LLM call. Produces grounded draft answer from relevant_chunks only. Writes to draft_answer.
verify
LLM call. Checks: is the answer grounded? Does it address the query? Routes to END or loops to search with feedback.
pythonagent/graph.py
from langgraph.graph import StateGraph, END
from agent.state import AgentState
from agent.nodes import plan, search, fetch, classify, synthesize, verify
from audit.tracer import traced  # our audit decorator

def route_after_verify(state: AgentState) -> str:
    """Conditional edge: loop back to search or terminate."""
    if state["verified"] or state["iteration"] >= 2:
        return "end"
    return "search"  # retry with verification_feedback in state

def build_graph() -> StateGraph:
    g = StateGraph(AgentState)

    # Register nodes — each wrapped by traced() at definition time
    g.add_node("plan",      traced(plan))
    g.add_node("search",    traced(search))
    g.add_node("fetch",     traced(fetch))
    g.add_node("classify",  traced(classify))
    g.add_node("synthesize",traced(synthesize))
    g.add_node("verify",    traced(verify))

    # Fixed edges
    g.set_entry_point("plan")
    g.add_edge("plan",      "search")
    g.add_edge("search",    "fetch")
    g.add_edge("fetch",     "classify")
    g.add_edge("classify",  "synthesize")
    g.add_edge("synthesize","verify")

    # Conditional edge: verify decides whether to loop or end
    g.add_conditional_edges("verify", route_after_verify, {
        "search": "search",
        "end":    END
    })

    return g.compile()
04 Audit layer — the tracer decorator

The traced() decorator is the entire audit layer in one place. It wraps any LangGraph node function, captures the before/after state, computes a diff, records tool calls, and persists a span to Postgres. Anomaly detection runs as a post-step after each span is written.

This design means the agent nodes themselves stay clean — no audit logic inside them. The assurance layer is a concern-separated wrapper, not tangled into business logic.

pythonaudit/tracer.py
import time, uuid, json
from functools import wraps
from typing import Callable
from deepdiff import DeepDiff       # pip install deepdiff
from audit.db import get_db
from audit.anomalies import detect_anomalies
from agent.state import AgentState

# Thread-local store for token counts injected by LLM callbacks
_token_counter = {}

def traced(node_fn: Callable) -> Callable:
    """Wrap a LangGraph node with full audit tracing."""

    @wraps(node_fn)
    async def wrapper(state: AgentState) -> AgentState:
        span_id   = str(uuid.uuid4())
        run_id    = state["run_id"]
        node_name = node_fn.__name__
        started   = time.time()

        # Deep-copy input state for diffing later
        input_snapshot = json.loads(json.dumps(state, default=str))

        try:
            output_state = await node_fn(state)
            success = True
            error   = None
        except Exception as e:
            output_state = state   # return unchanged state on failure
            success = False
            error   = str(e)

        duration_ms   = int((time.time() - started) * 1000)
        output_snapshot = json.loads(json.dumps(output_state, default=str))

        # Compute state diff — what did this node actually change?
        diff = DeepDiff(input_snapshot, output_snapshot,
                        ignore_order=True).to_dict()

        # Grab sequence order from DB (how many spans exist for this run so far)
        async with get_db() as db:
            seq = await db.fetchval(
                "SELECT COUNT(*) FROM spans WHERE run_id = $1", run_id)

            await db.execute("""
                INSERT INTO spans
                  (span_id, run_id, node_name, sequence_order,
                   started_at, duration_ms, input_state,
                   output_state, state_diff, success, error_message)
                VALUES ($1,$2,$3,$4,
                        to_timestamp($5), $6, $7::jsonb,
                        $8::jsonb, $9::jsonb, $10, $11)
            """,
            span_id, run_id, node_name, seq,
            started, duration_ms,
            json.dumps(input_snapshot),
            json.dumps(output_snapshot),
            json.dumps(diff),
            success, error)

        # Run anomaly detection against this span + run history
        await detect_anomalies(run_id, span_id, node_name,
                                input_snapshot, output_snapshot,
                                diff, duration_ms)

        return output_state

    return wrapper
05 PostgreSQL schema

Four tables. agent_runs is the top-level record. spans is the core audit log — one row per node execution. tool_calls stores individual tool invocations within a span. anomalies stores flagged deviations. All state snapshots stored as JSONB for queryability.

sqldb/schema.sql
CREATE EXTENSION IF NOT EXISTS "pgcrypto";

-- ── agent_runs ───────────────────────────────────────────────────────────
CREATE TABLE agent_runs (
    run_id          UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    query           TEXT         NOT NULL,
    status          VARCHAR(20)  NOT NULL DEFAULT 'running',
                                 -- 'running' | 'completed' | 'failed' | 'anomalous'
    started_at      TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    completed_at    TIMESTAMPTZ,
    total_tokens    INTEGER,
    final_answer    TEXT,
    iteration_count INTEGER      NOT NULL DEFAULT 0,
    anomaly_count   INTEGER      NOT NULL DEFAULT 0
);

-- ── spans ────────────────────────────────────────────────────────────────
CREATE TABLE spans (
    span_id         UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    run_id          UUID         NOT NULL REFERENCES agent_runs(run_id) ON DELETE CASCADE,
    node_name       VARCHAR(50)  NOT NULL,
    sequence_order  INTEGER      NOT NULL,  -- 0-indexed step number within the run
    started_at      TIMESTAMPTZ  NOT NULL,
    duration_ms     INTEGER      NOT NULL,
    input_state     JSONB        NOT NULL,  -- full state snapshot at node entry
    output_state    JSONB        NOT NULL,  -- full state snapshot at node exit
    state_diff      JSONB        NOT NULL,  -- DeepDiff output — what changed
    tokens_used     INTEGER,
    success         BOOLEAN      NOT NULL DEFAULT TRUE,
    error_message   TEXT,
    UNIQUE (run_id, sequence_order)
);

-- ── tool_calls ───────────────────────────────────────────────────────────
CREATE TABLE tool_calls (
    call_id         UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    span_id         UUID         NOT NULL REFERENCES spans(span_id) ON DELETE CASCADE,
    run_id          UUID         NOT NULL,  -- denormalised for fast run-level queries
    tool_name       VARCHAR(50)  NOT NULL,  -- 'web_search' | 'fetch_page' | 'llm_call'
    called_at       TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    duration_ms     INTEGER,
    input_args      JSONB,        -- what was passed to the tool
    output_summary  JSONB,        -- truncated output (first 500 chars per field)
    tokens_used     INTEGER,      -- for LLM calls only
    success         BOOLEAN      NOT NULL DEFAULT TRUE,
    error_message   TEXT
);

-- ── anomalies ────────────────────────────────────────────────────────────
CREATE TABLE anomalies (
    anomaly_id      UUID         PRIMARY KEY DEFAULT gen_random_uuid(),
    run_id          UUID         NOT NULL REFERENCES agent_runs(run_id) ON DELETE CASCADE,
    span_id         UUID         REFERENCES spans(span_id),  -- null = run-level anomaly
    anomaly_type    VARCHAR(50)  NOT NULL,
    severity        VARCHAR(10)  NOT NULL,  -- 'low' | 'medium' | 'high'
    description     TEXT         NOT NULL,
    detected_at     TIMESTAMPTZ  NOT NULL DEFAULT NOW(),
    evidence        JSONB         -- raw data that triggered the flag
);

-- ── indexes ──────────────────────────────────────────────────────────────
CREATE INDEX idx_spans_run_id    ON spans(run_id);
CREATE INDEX idx_spans_node      ON spans(node_name);
CREATE INDEX idx_tool_calls_span ON tool_calls(span_id);
CREATE INDEX idx_anomalies_run   ON anomalies(run_id);
CREATE INDEX idx_spans_input     ON spans USING GIN (input_state);   -- JSONB search
CREATE INDEX idx_spans_diff      ON spans USING GIN (state_diff);

-- ── dashboard view ───────────────────────────────────────────────────────
CREATE VIEW run_summary AS
    SELECT
        r.run_id,
        r.query,
        r.status,
        r.started_at,
        EXTRACT(EPOCH FROM (r.completed_at - r.started_at)) * 1000 AS total_ms,
        r.iteration_count,
        r.anomaly_count,
        COUNT(s.span_id)                                            AS span_count,
        SUM(s.tokens_used)                                          AS total_tokens,
        LEFT(r.final_answer, 120)                                   AS answer_preview
    FROM agent_runs r
    LEFT JOIN spans s ON s.run_id = r.run_id
    GROUP BY r.run_id;
06 Anomaly detection rules

Six rules. Each runs after every span is written. Rules produce rows in the anomalies table and increment agent_runs.anomaly_count. When anomaly_count > 0 the run status is updated to anomalous — surfaced prominently in the dashboard.

HIGH
unexpected_loop
Same node_name appears more than 2 times in the spans table for this run_id. Indicates the agent is stuck — verify is failing to converge. Evidence: list of repeated span_ids.
HIGH
stale_state
state_diff is empty (DeepDiff returned {}) — node executed but made no change to state. Indicates silent failure: the node ran but its outputs were not written. Evidence: the empty diff object.
HIGH
empty_required_output
Node-specific: plan produced zero sub_queries; classify produced zero relevant_chunks; synthesize produced empty draft_answer. Per-node output contract violated. Evidence: the field that was empty and node name.
MED
token_spike
tokens_used for this span is more than 2× the rolling average for the same node_name across the last 20 runs. Could indicate prompt injection inflating the context or runaway generation. Evidence: current tokens, rolling average, ratio.
MED
slow_node
duration_ms exceeds 3× the rolling p50 for the same node_name. Indicates an external tool call hanging or an unusually large LLM response. Evidence: duration, rolling p50, node name.
LOW
low_relevance_pass
classify node passed fewer than 2 chunks with score ≥ 0.5. Synthesize will be working with weak evidence — final answer reliability is low. Evidence: chunk scores, number kept vs total.
pythonaudit/anomalies.py
async def detect_anomalies(run_id, span_id, node_name,
                             input_state, output_state,
                             diff, duration_ms):
    async with get_db() as db:
        flags = []

        # Rule 1: unexpected_loop
        count = await db.fetchval(
            "SELECT COUNT(*) FROM spans WHERE run_id=$1 AND node_name=$2",
            run_id, node_name)
        if count > 2:
            flags.append({"type": "unexpected_loop", "severity": "high",
                "desc": f"{node_name} visited {count} times in run",
                "evidence": {"count": count}})

        # Rule 2: stale_state
        if not diff:
            flags.append({"type": "stale_state", "severity": "high",
                "desc": f"{node_name} made no state change",
                "evidence": {"diff": {}}})

        # Rule 3: empty_required_output (per-node contracts)
        contracts = {
            "plan":       ("sub_queries",       lambda v: len(v) > 0),
            "classify":   ("relevant_chunks",   lambda v: len(v) > 0),
            "synthesize": ("draft_answer",      lambda v: bool(v.strip())),
        }
        if node_name in contracts:
            field, check = contracts[node_name]
            val = output_state.get(field)
            if val is None or not check(val):
                flags.append({"type": "empty_required_output", "severity": "high",
                    "desc": f"{node_name} produced empty {field}",
                    "evidence": {"field": field, "value": val}})

        # Rule 4: token_spike
        tokens = output_state.get("_tokens_this_node", 0)
        avg = await db.fetchval("""
            SELECT AVG(tokens_used) FROM (
              SELECT tokens_used FROM spans
              WHERE node_name=$1 AND tokens_used IS NOT NULL
              ORDER BY started_at DESC LIMIT 20
            ) t""", node_name) or 0
        if avg > 0 and tokens > avg * 2:
            flags.append({"type": "token_spike", "severity": "medium",
                "desc": f"Token use {tokens} vs avg {avg:.0f} for {node_name}",
                "evidence": {"tokens": tokens, "rolling_avg": round(avg)}})

        # Write all flags and update run status
        for f in flags:
            await db.execute("""
                INSERT INTO anomalies
                  (run_id, span_id, anomaly_type, severity, description, evidence)
                VALUES ($1,$2,$3,$4,$5,$6::jsonb)""",
                run_id, span_id, f["type"], f["severity"],
                f["desc"], json.dumps(f["evidence"]))
            await db.execute("""
                UPDATE agent_runs
                SET anomaly_count = anomaly_count + 1, status = 'anomalous'
                WHERE run_id = $1""", run_id)
07 FastAPI — audit query layer

Five endpoints. These power the dashboard and serve as the public interface to the trace store — analogous to an observability API.

pythonapi/routes.py
GET  /runs                    # list recent runs (from run_summary view)
GET  /runs/{run_id}           # full run record + all spans in order
GET  /runs/{run_id}/spans     # spans only, ordered by sequence_order
GET  /runs/{run_id}/anomalies # all anomalies for a run
GET  /spans/{span_id}/diff    # the state_diff for a single span (human-readable)
